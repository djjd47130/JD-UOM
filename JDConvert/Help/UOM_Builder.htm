Summary

(NOT YET FULLY SUPPORTED)

This mode allows you to create your own custom UOMs. These are saved outside of the built-in system UOMs, and are implemented throughout the system in the same manner. Custom UOMs are saved per Windows user.

 

Custom UOM List

This list contains all user-created UOMs. Selecting an item will display its details down below.

There are several columns in this list:

  • Custom UOM Name - The main SINGULAR name of the UOM, which is its main unique identifier.
  • Type - The type of custom UOM entry. See "Custom UOM Types" below.
  • Category - The overall category associated with the UOM.
  • Suffix - The text suffix to display after a value of the UOM, which also acts as a case-sensitive unique identifier.
  • Systems - The UOM system(s) associated with the UOM, separated by commas.

 

Custom UOM Types

Any given custom UOM must be assigned one of the following types:

Metric

Multiple metric-specific UOMs will be automatically generated for each of the selected Metric units.

  • Category (Required ) - The overall category associated with the UOM (Distance, Area, Volume, Mass, etc.).
  • Name (Singular) (Required ) - The SINGULAR name of the UOM. Used as the unique identifier across the application.
    • NOTE : This is automatically used in combination with all selected Metric Units. For example, the base name "Shaq" will produce UOMs such as "Millishaq", "Centishaq", etc.
    • NOTE : Each UOM generated will automatically append the singular name with an "s" to make the plural version.
  • Base UOM (Optional ) - Specifies which of the generated UOMs to use as the base.
    • Leave blank to let the application automatically use the Metric base.
  • Suffix (Required ) - The text to be displayed after a given UOM value.
    • NOTE : This is automatically used in combination with all the selected Metric Units and their suffix. For example, the base  suffix "Shaq" will produce UOMs with suffixes such as "mShaq", "cShaq", etc.
  • Metric Units (Required) - Specifies each of the possible Metric units to be auto-generated. A separate UOM will be created for each option you check. Must select at least one.

Simple

A single UOM with a simple "Factor" to be used to convert to and from the category's base UOM.

  • Category (Required ) - The overall category associated with the UOM (Distance, Area, Volume, Mass, etc.).
  • Category Base (Optional ) - Whether this UOM is to be used as the base UOM for the given category.
    • NOTE: BE CAREFUL WITH THIS, ESPECIALLY WHEN USING EXISTING BUILT-IN CATEGORY. This will override whatever base is already assigned to that category.
  • Name (Singular)(Required)  - The SINGULAR name of the UOM. Used as the unique identifier across the application.
  • Name (Plural) (Required ) - The PLURAL name of the UOM. Loosely used as a unique identifier across the application.
  • Suffix (Required ) - The text value to be displayed after a value of the UOM. Used as a case-sensitive unique identifier across the application.
  • System(s) (Required ) - One or more UOM systems associated with the UOM (Metric, Imperial, US Customary, etc.).
  • Factor (Required ) - The conversion factor to be used to calculate between this UOM and the category's base UOM.
    • For example, in the "Distance" category, where "Meter" is the base, this number represents how much of the base is within this UOM.
    • As pictured above, 1 "Shaq" is equal to 2.158999 "Meters".

Formula

A single UOM with string-based mathematical expressions to convert from and to the category's base UOM.

  • Category (Required ) - The overall category associated with the UOM (Distance, Area, Volume, Mass, etc.).
  • Category Base (Optional ) - Whether this UOM is to be used as the base UOM for the given category.
    • NOTE: BE CAREFUL WITH THIS, ESPECIALLY WHEN USING EXISTING BUILT-IN CATEGORY. This will override whatever base is already assigned to that category.
  • Name (Singular) (Required ) - The SINGULAR name of the UOM. Used as the unique identifier across the application.
  • Name (Plural) (Required ) - The PLURAL name of the UOM. Loosely used as a unique identifier across the application.
  • Suffix (Required ) - The text value to be displayed after a value of the UOM. Used as a case-sensitive unique identifier across the application.
  • System(s)(Required)  - One or more UOM systems associated with the UOM (Metric, Imperial, US Customary, etc.).
  • From Base(Required)  - The formula to be used to convert this UOM FROM the category's base UOM.
  • To Base (Required ) - The formula to be used to convert this UOM TO the category's base UOM.
  • NOTE : The formulas must use the text "Value" to represent the value being converted.

 

Editing Buttons

On the right side of the UOM details area, there are a number of buttons for editing custom UOMs:

  •  - Add new custom UOM
  • - Edit selected custom UOM
  • - Delete selected custom UOM
  • - Save changes to custom UOM
  • - Cancel changes to custom UOM
